# DNS & ICMP Network Traffic Analysis

## Project Overview
This project analyzes DNS and ICMP traffic captured using tcpdump during a simulated cybersecurity incident.

## Objective
Identify the network protocol affected and determine the root cause of website accessibility issues.

## Skills Demonstrated
- Packet Analysis
- DNS Investigation
- ICMP Troubleshooting
- Incident Reporting
- Network Security Fundamentals
- TCP/IP Model Analysis

## Tools Used
- tcpdump
- DNS
- UDP
- ICMP

## Key Findings
- DNS queries were sent using UDP.
- DNS server returned ICMP "udp port 53 unreachable" messages.
- DNS resolution failed.
- Root cause likely involved DNS service outage, firewall blocking, or misconfiguration.

## Outcome
Successfully identified DNS service failure through packet analysis and documented findings in a professional incident report.
